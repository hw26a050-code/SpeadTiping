# UI・デザイン方針仕様書

## 1. デザインコンセプト
- **雰囲気**: 初心者向けの柔らかい、親しみやすい雰囲気。
- **トーン＆マナー**: 明るくシンプルな構成。パステルカラー（暖色系など）を基調とし、丸みを帯びたやわらかいデザイン要素を使用します。

## 2. 画面レイアウト構成

### A. タイトル（メイン）画面
- **タイトルロゴ**: 柔らかく親しみやすいフォントのタイトル表示。
- **設定（オプション）エリア**:
  - プレイヤーは難易度を選択する前に、以下の設定を切り替えることができます。
    - **仮想キーボード表示**: ON / OFF（初心者向けに、画面上に次に押すキーを示すガイドキーボードを表示するかどうか）
    - **サウンド**: ON / OFF（打鍵音や正解・失敗時の効果音、BGMの有無）
- **難易度選択・スタートボタン**:
  - 「Easy」「Normal」「Hard」の各難易度ボタンを配置し、難易度を選択するとゲームプレイ画面へ遷移します。

### B. ゲームプレイ画面
- **ステータス表示エリア**:
  - **現在のスコア**: これまでに正解した単語数（例：「Score: 12」）
  - **残りライフ**: ハートマーク（初期値3）でビジュアル表示。
  - **制限時間ゲージ**: 各単語の制限時間（ひらがな文字数 * 1.5秒）に合わせた、滑らかに減少するバー。
- **出題エリア（中央）**:
  - **メインの出題単語**: 「漢字（例：林檎）」で表示。
  - **ふりがな**: 漢字のすぐ上に小さめの文字で「ひらがな（例：りんご）」を表示。
  - **入力対象ローマ字**: 
    - 入力済みの文字（灰色や薄い緑など）と、未入力の文字（はっきりした黒や青など）が視覚的に色分けされて表示される。
    - ミスした瞬間は、一瞬画面が赤くフェードするなどの柔らかいミス演出。
- **仮想キーボード（表示設定がONの場合のみ、下部に配置）**:
  - 次に入力すべきキーを視覚的にハイライトし、タイピング初心者の入力をアシストします。

### C. ゲームオーバー（リザルト）画面
- **ゲームオーバー演出**: 終了時にライフが0、または制限時間切れ（Hard時）になったことを知らせるマイルドな演出。
- **結果発表**:
  - プレイした難易度、および最終スコア（正解した単語数）を大きく表示。
- **操作ボタン**:
  - **もう一度遊ぶ（リトライ）**: 同じ難易度、同じ設定で即座にプレイを再開。
  - **タイトルに戻る**: 難易度や設定を再調整できるタイトル画面へ戻る。

### D. レスポンシブとスクロール制御方針
- **スクロールバーの排除**: 画面全体を `h-screen` および `100dvh` に収めることで、ブラウザ全体のスクロールバー（スライドバー）が表示されないように制御します。
- **コンパクト設計**: 各要素の余白（Padding/Margin）やフォントサイズ、ロゴの大きさをコンパクトに調整し、標準的なラップトップやモバイルの全画面表示においてスクロールなしで完全にフィットするようにします。
- **コンテンツ領域の個別スクロール**: 極端に画面の高さが低い環境でも、ヘッダー・フッターなどのレイアウト枠は固定し、メイン領域（`<main>`）の内部のみが滑らかにスクロール（`overflow-y-auto`）するように設計します。
